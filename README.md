# 宵待ち麻雀倶楽部 Ver.2 (四人打ち・試作版)

*今夜、最後の一局を。*

二人打ち試作版をベースに、**日本式の4人麻雀(東風戦)** として再設計したVer.2です。
二人打ち版のコードを場当たり的に拡張するのではなく、ゲーム状態・ターン進行・
鳴き処理・点数計算を4人前提で作り直しています。

- プレイヤー(画面下) + CPU3名(左・上・右)
- 上家(左)=攻撃型CPU / 対面(上)=神楽坂 綾乃(バランス型) / 下家(右)=守備型CPU
- 登場人物はすべて20歳以上です

## 麻雀牌のビジュアル

牌34種(萬子・索子・筒子の各1〜9 + 字牌7種)は牌一覧シート画像から切り出した
PNGを表示に使用しています。

- 元画像: `assets/source/mahjong-tile-sheet.png`(数牌27種) /
  `assets/source/honor-tile-sheet.png`(字牌7種)
- 切り出し済み: `assets/tiles/faces/{m,s,p}{1-9}.png` + `z{1-7}.png`
  (各256×352・背景透明)。z1-z7 = 東南西北白發中
- 切り出し方法: .NET(System.Drawing)による自動処理。グリッド検出 → 牌ごとに
  クロップ → 縁の背景色から連結する画素だけを flood fill で透明化(牌内部の
  緑=索子の竹・一索の鳥は非連結なので保護)→ 全牌を同一の本体サイズ枠へ正規化
- **白牌(z5)は完全な無地**(アイボリー面のみ)
- 裏面(CPU手牌・暗槓・山)は従来どおりCSSの竹牌風
- 画像読み込み失敗時は自動でSVG牌(`faceSVG`)にフォールバック
- 確認用ページ: `dev-tile-catalog.html`(34種+裏面を一覧表示、市松背景で透明化を検証)

牌サイズは `css/tiles.css` のCSS変数(`--player-tile-*` / `--river-tile-*` /
`--cpu-tile-*` ほか)で一元管理し、同一PNGをCSSで縮小表示しています。

## 起動方法

静的サイトです。ローカルサーバ経由での起動を推奨します。

- PowerShell: `powershell -ExecutionPolicy Bypass -File serve.ps1` → http://localhost:8420/
- Python がある場合: `python -m http.server 8420`
- `index.html` を直接開いても動作します(Chrome / Edge 推奨)

推奨解像度: 1280×720 以上のPCブラウザ。スマートフォンでは案内画面が表示されます。

## キャラクター選択と追加方法

GAME START後、登録済みキャラクターからCPUを3人選択します。選択順が右席・対面・左席の順になります。「おまかせ」では重複なしで3人をランダム選択します。

現在は神楽坂綾乃、有栖川リリ、謎の覆面、もふ蔵、田辺一、梅沢トメの6人を登録しています。

キャラクターは `data/characters.js` の `YM.CHARACTERS` と `YM.CHARACTER_ORDER` で管理します。人数は6人に固定しておらず、定義を追加すると選択画面のカードも自動生成されます。ゲーム状態・席順・セーブデータでは表示名ではなく一意の `id` を使用します。

画像は `assets/characters/{id}/` に表情・イベント名で整理しています。`data/characters.js` の `images` へ用途別キーを登録すると、通常カード・開始挨拶・イベント画面で自動的に使い分けられます。画像が読み込めない場合も代替表示でゲームを続行できます。

## 対局画面レイアウト(2026-07-24 俯瞰型へ全面刷新)

対局画面を「正方形の卓を真上から見る」完全俯瞰型へ作り直しました。
旧レイアウト(卓面73% + 右情報欄27%の3段構成)は廃止し、
`css/table-split.css` は `css/table-overhead.css` に置き換えています。

### 使用素材

- `assets/ui/table/taku3.png` (941x941・正方形) … 卓面。1枚をそのまま敷く
- `assets/ui/table/takucenter.png` (256x243) … 卓中央の装飾。内側の黒地へ
  局数・本場・供託・残り山・ドラを収める

### レイアウトの決まりごと

- 卓の1辺 `--table-size` が唯一の基準寸法。以降の座標はすべてこの比率で決まる
- 河は中央装飾の外周4方向。常に6列x3段=18枚ぶんの領域を確保するので、
  段が増えても河の位置は動かない
- 副露は卓の四隅(各家の右手前)。牌は席の向きに合わせて回転
- CPUの手牌は卓の内周(上辺・左辺・右辺)に沿わせる
- 4人の情報カードは卓の左右の余白帯へ。卓面の情報を隠さない
- 自分の手牌とアクションバーは卓の下の帯に独立して配置

### 実装上の注意(はまりどころ)

- **% の padding / width は使わない**。`#table-player-cards` が `inset:0`
  (フレーム全幅)なので、カード内で `36%` と書くと約570pxに化けてカードが
  破綻する。カード内の寸法は必ず `--card-w` を基準に `calc()` で出すこと
- 絶対配置のカードは `left`/`right` の**片側だけ**を指定する。両方が確定すると
  箱が両端まで引き伸ばされ、`width` の指定が無視される
- `#game-frame` は5pxの枠線を持つため、`--table-size` の計算では
  `--frame-border`(10px)を引いている
- 4つの河は風車状に並ぶため対角どうしが角で触れやすい。`--river-gap` に
  下限(6px)を設けて回避している
